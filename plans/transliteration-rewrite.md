# Transliteration Engine Rewrite: Latin-in-field, Dictionary-Ranked Commit

> Execution note: per CLAUDE.md's planning convention, copy this file to
> `plans/transliteration-rewrite.md` as the first step of execution.

## Context

The current Amharic pipeline shows the **greedy fidel reading live in the field's composing
region** while mirroring the raw Latin to a preview strip. This has proven buggy and structurally
limited: the greedy reading is often the wrong word ("fkr" shows ፍክር when the user means ፍቅር),
alternate readings are generated by uniformly flipping *every* consonant to its Nth alternate
(so mixed choices like "t primary + k alternate" are never offered), and the in-field fidel plus
discard-on-exit plus fidel→Latin resume history form a fragile web of state.

The rewrite inverts the model to the proven pinyin-IME pattern (agreed with the user):

1. **The field shows the raw Latin** (underlined composing region) while typing. Fidel readings
   appear only in the suggestion strip. **Space commits the first suggestion.**
2. **Cursor-aware**: moving the caret onto a committed Amharic word resumes composition there so
   suggestions work at the new position.
3. **Greedy digraphs keep structural priority**: "sh" ranks ሽ above ስህ.
4. **Dictionary-aware ranking**: when a non-greedy reading has an exact dictionary match and the
   greedy one doesn't ("fkr" → ፍቅር exists, ፍክር doesn't), the matching reading is promoted to
   first, so space inserts it.

Decisions confirmed with the user:
- **Exit rule**: leaving the word without space/enter/punctuation/chip (keyboard hidden, caret
  tapped away) **deletes the tentative Latin** — keeps the existing "nothing commits without an
  accept gesture" contract (auto-commit was tried and reverted twice; see WordComposer docs).
- **Top candidate display**: first chip of the suggestion strip, **visually highlighted**;
  `BufferPreviewStrip` is removed (the Latin it mirrored is now in the field itself).
- **Ranking without an exact match**: structural (greedy) order stays first — deterministic.
  Exact-match promotion is the *only* dictionary reordering. Among multiple exact matches,
  structural order also wins (the Amharic dictionary's frequencies are heuristic, not real).

English mode is untouched behaviorally.

## Architecture

```
AmharicTable (data, mostly unchanged)
   └─ Transliterator (rewritten candidate engine, pure Kotlin)
        ├─ transliterate(latin)           — greedy reading (kept: key previews, punctuation, fallback)
        └─ candidates(latin, limit)       — lattice-based ranked readings, greedy first  [NEW]
   └─ AmharicWordReverser                 — fidel → Latin for caret resume  [NEW, pure Kotlin]
WordTrie
   └─ frequencyOf(word): Int?             — exact-match lookup  [NEW]
CandidateRanker (pure Kotlin)             — stable partition: exact matches first  [NEW]
WordComposer                              — Latin composes inline; commit goes through a
                                            commitTransform lambda  [CHANGED]
AddiyonKeyboardService                    — wiring: suggestions, commit paths, resume  [CHANGED]
SuggestionBar / KeyboardScreen            — highlighted first chip; strip removed  [CHANGED]
```

## Step-by-step

### 1. Candidate engine — rewrite `Transliterator.readings()` as a segmentation/family lattice
`app/src/main/java/com/addiyon/keyboard/transliteration/Transliterator.kt`

Keep the stateless whole-buffer design and `transliterate()` (greedy) exactly as-is — it still
serves key corner previews, punctuation pass-through, and the no-dictionary fallback.

Replace `readings()` / `transliterateSplit()` / `bufferHasAlternates()` with:

- **Parse into a unit lattice.** Walk the buffer with the same matching rules as
  `forEachUnit` (longest consonant case-sensitive → case-insensitive; vowel; bare vowel;
  pass-through), but at each digraph position also record the **split** segmentation as an
  alternative branch (per-digraph, not the current all-or-nothing global split). Each unit
  carries its ordered list of renderings: primary family form first, then each alternate from
  `AmharicTable.consonantAlternates` (and the glottal↔pharyngeal flip for bare vowels).
- **`candidates(latin: String, limit: Int = 24): List<String>`** — bounded beam over the
  lattice product, ordered by structural score so index 0 is always the greedy reading:
  segmentation greediness dominates (combined digraph before split — requirement 3), then
  per-unit family rank (primary before 1st alternate before 2nd), summed left-to-right; ties
  broken by unit position (earlier units flip last). Dedupe. Cap ~24 candidates — enough to
  contain any realistic exact-match reading without combinatorial blowup on long words.
- This fixes the mixed-alternates hole by construction: "fkr" yields ፍክር, ፍቅር, ...;
  a buffer needing "t primary + k alternate" now gets that combination too.

`AmharicTable` needs no structural change (families, alternates, bare vowels, multiTapCycles,
punctuation all stay). Only additions if the lattice wants a helper view of per-spelling
alternates — reuse `consonantAlternates` as-is.

### 2. Exact-match lookup — `WordTrie.frequencyOf`
`app/src/main/java/com/addiyon/keyboard/suggestion/WordTrie.kt`, plus a passthrough on
`WordDictionary` (`app/src/main/java/com/addiyon/keyboard/suggestion/WordDictionary.kt`).

Walk the path for the (lowercased) word, return the terminal node's frequency or null. For
fidel the lowercasing is a no-op. O(word length), called ≤ 24 times per keystroke — negligible.

### 3. Ranking — new pure `CandidateRanker`
New file `app/src/main/java/com/addiyon/keyboard/suggestion/CandidateRanker.kt` (pure Kotlin,
JVM-testable):

```kotlin
fun rank(candidates: List<String>, isWord: (String) -> Boolean): List<String>
```

A **stable partition**: candidates that are exact dictionary words first, everything else after,
each group keeping the engine's structural order. That is the entire dictionary influence on
the commit target — deterministic, no frequency reordering (per user decision). Covers both
requirements 3 and 4:
- "sh": neither ሽ nor ስህ is a word → structural order → ሽ first.
- "fkr": ፍቅር is a word, ፍክር isn't → ፍቅር promoted to first.

### 4. `WordComposer` — Latin composes inline; commit through a transform
`app/src/main/java/com/addiyon/keyboard/composing/WordComposer.kt`

- Drop the `render` parameter (both languages now compose the raw buffer verbatim;
  `display == raw`). `pushComposing()` pushes `raw`.
- Add `commitTransform: (String) -> String = { it }`: **every path that turns the buffer into
  committed field text** goes through it — `commit()`, and the resumed-word restore branches of
  `finish()`/`abandon()`. Amharic's transform (wired in the service) returns the top-ranked
  candidate for the buffer, falling back to `Transliterator.transliterate(raw)` when the
  dictionary isn't loaded or ranking returns nothing.
  - Recompute at commit time from the current buffer (don't cache the last strip state) so the
    committed word can never be stale relative to the buffer.
- Keep `discardOnExit = true` semantics for Amharic: non-resumed `finish()`/`abandon()` still
  delete the composing region (now deleting Latin instead of fidel — the user-confirmed exit
  rule). The `resumed` exemption stays, now committing `commitTransform(raw)` so restored words
  land as fidel, never as Latin.
- `commit()` reports `onCommit(raw, committedText)` where `committedText` is the transform
  output — the fidel→Latin history keeps working unchanged.
- `commitSuggestion(word)` unchanged (chip tap replaces the composing Latin with the chip).

### 5. Service wiring
`app/src/main/java/com/addiyon/keyboard/AddiyonKeyboardService.kt`

- **Composer construction**: `amharicComposer` loses `render`, gains
  `commitTransform = { raw -> topCandidate(raw) }` where
  `topCandidate(raw) = rank(Transliterator.candidates(raw), dictionary::isWord).firstOrNull()
  ?: Transliterator.transliterate(raw)`.
- **`amharicSuggestions()`**: compute `ranked = rank(candidates(latin), ...)`. The strip now
  shows ranked readings **including the first one** (the field shows Latin, so nothing is
  redundant anymore — delete the `.drop(1)`). Then, per ranked reading in order, append
  prefix completions from the dictionary; then the existing fuzzy pass (reuse the current
  `fidelSubstitutionCost` machinery unchanged); dedupe; cap at `AMHARIC_SUGGESTION_LIMIT`.
- **Remove `amharicBufferLatin`** and its `updateSuggestions()` bookkeeping — the preview strip
  is gone.
- **Commit paths** (`onSpace`, `onEnter`, non-word `onCharacter` incl. ፣/።, `toggleLanguage`,
  `toggleNumberMode`, `openEmojiPanel`, voice start): no signature changes — they all call
  `activeComposer.commit()`, which now routes through `commitTransform`. Space therefore
  inserts the first suggestion + " " (requirement 1).
- **Multi-tap** (`applyMultiTap`/`startMultiTap`): mechanically unchanged (it edits the raw
  buffer, which is the composing text). The field now visibly shows SERA spellings the cycle
  substitutes (e.g. "`a", "H") — acceptable: the buffer is tentative and the strip shows the
  fidel result. The skip-step comparison keeps using `Transliterator.transliterate`.

### 6. Cursor-aware resume (requirement 2)
Same file. The `onUpdateSelection` → debounced `resumeAtCursorRunnable` machinery stays as-is
(abandon on walk-away, resume when the caret settles). Two changes on the Amharic branch:

- `resumeAmharicWordIfAny` keeps trying `amharicCommitHistory` first (exact raw Latin, free).
- **New fallback — `AmharicWordReverser`** (new pure-Kotlin file in `transliteration/`): build a
  reverse map once from `AmharicTable.families` — fidel char → canonical Latin spelling
  (primary family key + vowel spelling by form index; ua forms → key + "ua"). To resume an
  arbitrary committed fidel word (earlier session, other keyboard, paste): reverse each char,
  concatenate, then **round-trip verify** — accept only if
  `Transliterator.transliterate(reversed) == fidelWord`; otherwise skip the resume (fall
  through to a fresh word, exactly like today). The verification guard is what makes this safe
  despite reverse transliteration being ambiguous in general.
- Resumed words remain protected by the `resumed` flag: exiting restores them via
  `commitTransform` (fidel), never deletes them.

### 7. UI
- `ui/SuggestionBar.kt`: in Amharic mode, render the **first chip highlighted** (tinted /

- `ui/KeyboardScreen.kt`: remove the `BufferPreviewStrip` slot; **delete

- Key corner previews already call `Transliterator.transliterate` live — no change.

## Risks

- **Combinatorial growth** of candidates on long words → hard cap + beam by structural score;
  exact-match checks are O(len) each.
- **Stale commit target**: mitigated by recomputing the top candidate inside `commitTransform`
  at commit time rather than caching strip state.
- **Resume flip UX**: when the caret lands on a committed fidel word it visibly turns back into
  underlined Latin. Consistent with "typing = Latin", and exiting restores the fidel — but
  worth watching in manual testing.
- **Dictionary not yet loaded** at first keystrokes: ranking degrades to structural order and
  commit falls back to the greedy reading — never blocks typing.
- **Multi-tap SERA spellings visible in-field** ("`a", "H"): accepted trade-off; revisit only
  if it tests badly.

## Verification (end-to-end, on emulator)

After `installDebug`, enable/switch to Addiyon Keyboard and check in a plain text field:
1. Type `selam` — field shows underlined "selam"; first chip highlighted ሰላም; **space** →
   "ሰላም " committed, buffer clear.
2. Type `fkr` — first chip is **ፍቅር** (exact match promoted over greedy ፍክር); space inserts it.
3. Type `sh` — field shows "sh"; chips ordered ሽ then ስህ.
4. Type `abc`, hide the keyboard — the Latin disappears from the field (nothing committed).
5. Type `abc`, tap elsewhere in existing text — Latin removed, caret lands where tapped;
   if it lands at the end of a committed Amharic word, after ~150 ms the word flips to
   underlined Latin and suggestions appear (both for a word committed this session and one
   pasted in — the reverser fallback).
6. Multi-tap `a a` quickly — buffer cycles spellings, first chip cycles አ→ዓ→…
7. `,` and `.` — commit the word first, then ፣ / ።.
8. English mode: type/suggestions/backspace behave exactly as before.
